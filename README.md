# E‑Treasury Form 25

Web εφαρμογή για χειροκίνητη συμπλήρωση, επαναχρησιμοποίηση προτύπων ανά πεδίο, αυτόματο υπολογισμό και εκτύπωση/PDF του εντύπου Γ.Λ.25.

## Λειτουργίες MVP

- χειροκίνητη συμπλήρωση όλων των βασικών πεδίων,
- αποθήκευση, επιλογή και διαγραφή προτύπων ξεχωριστά για κάθε πεδίο,
- αυτόματη προσωρινή αποθήκευση της τρέχουσας φόρμας στον browser,
- αυτόματος υπολογισμός Φ.Π.Α. και συνολικού ποσού,
- μετατροπή του συνολικού ποσού σε ελληνικές λέξεις,
- ανεξάρτητη αρίθμηση τιμολογίων ανά υπάλληλο,
- προεπισκόπηση Α4 και εκτύπωση ή αποθήκευση σε PDF.

## Τοπικά δεδομένα και προφίλ υπαλλήλου

Η εφαρμογή παραμένει static frontend και όλα τα δεδομένα αποθηκεύονται τοπικά στον browser μέσω `localStorage`. Δεν υπάρχει server ή κεντρικός έλεγχος συγχρονισμού.

Κάθε browser profile έχει τοπικό προφίλ υπαλλήλου με σταθερό εσωτερικό `employeeId`, εμφανιζόμενο κωδικό υπαλλήλου και ονοματεπώνυμο. Ο κωδικός υπαλλήλου κλειδώνει μετά την πρώτη έκδοση τιμολογίου, ώστε να μην δημιουργείται νέα σειρά αλλάζοντας τον κωδικό.

Η εφαρμογή δεν μπορεί να ελέγξει αν ο ίδιος κωδικός υπαλλήλου χρησιμοποιείται σε άλλον υπολογιστή. Ο κωδικός πρέπει να είναι μοναδικός μέσα στην εκδούσα μονάδα με οργανωτικό έλεγχο.

## Αρίθμηση τιμολογίων ανά υπάλληλο

Η εσωτερική εκδούσα μονάδα χρησιμοποιεί `serviceId` / `issuerUnitId`:

`STATE-FAIR-MANAGEMENT`

Ο εμφανιζόμενος κωδικός εκδούσας μονάδας, όπως `ΥΕΕΒ-ΥΕ-ΚΔΧΚΕ`, χρησιμοποιείται στον πλήρη αριθμό τιμολογίου.

Κάθε υπάλληλος έχει ανεξάρτητη σειρά `00001` έως `99999`. Η σειρά δεν επανεκκινείται και δεν κάνει wrap. Μετά το `99999`, ο συγκεκριμένος υπάλληλος δεν μπορεί να εκδώσει άλλα τιμολόγια με τον ίδιο κωδικό.

Ο πλήρης αριθμός περιλαμβάνει κωδικό μονάδας, κωδικό υπαλλήλου και πενταψήφιο αριθμό:

`ΥΕΕΒ-ΥΕ-ΚΔΧΚΕ-ΜΜ / 00001`

Η μοναδικότητα τιμολογίου καθορίζεται από τον συνδυασμό `issuerUnitId + employeeId + invoiceNumber`. Δύο διαφορετικοί υπάλληλοι μπορούν να έχουν τον ίδιο πενταψήφιο αριθμό.

Κατά τη μετάβαση, παλαιά τιμολόγια χωρίς `employeeId` συνδέονται με το τοπικό προφίλ υπαλλήλου μετά από επιβεβαίωση. Παλαιά δεδομένα με `STATE-FAIR-SPACE-MANAGEMENT` μεταφέρονται στο `STATE-FAIR-MANAGEMENT` χωρίς επανεκκίνηση της αρίθμησης.

## Προσωπικό archive

Κάθε υπάλληλος πρέπει να κατεβάζει τακτικά το προσωπικό του archive. Το JSON export είναι πλήρες backup των snapshots του ενεργού υπαλλήλου. Το CSV export είναι αναφορά για έλεγχο και ανάγνωση, όχι πλήρες backup.

Η λειτουργία `Λήψη του αρχείου μου` δημιουργεί προσωπικό JSON backup που περιλαμβάνει metadata εκδούσας μονάδας, το σταθερό `employeeId`, την κατάσταση αρίθμησης και τα πλήρη snapshots. Η λειτουργία `Επαναφορά προσωπικού αρχείου` δέχεται μόνο αυτό το JSON backup· αρχεία CSV δεν εισάγονται και δεν θεωρούνται αντίγραφα ασφαλείας.

Κατά την επαναφορά, η εφαρμογή ελέγχει πρώτα ολόκληρο το αρχείο, εμφανίζει περίληψη και ζητά επιβεβαίωση πριν γράψει στο `localStorage`. Το restore διατηρεί το ίδιο `employeeId`, επαναφέρει τον κωδικό και το όνομα υπαλλήλου από το backup και συνεχίζει την αρίθμηση από το υψηλότερο ασφαλές σημείο. Αν ο browser έχει ήδη δεδομένα άλλου `employeeId`, η εφαρμογή δεν αντικαθιστά αυτόματα το υπάρχον προσωπικό profile.

Η εφαρμογή χρησιμοποιεί αποκλειστικό lock κατά την έκδοση τιμολογίου, ώστε δύο καρτέλες να μην εκδώσουν τον ίδιο αριθμό. Όταν το Web Locks API δεν είναι διαθέσιμο και εντοπιστεί δεύτερη ενεργή καρτέλα, η έκδοση μπλοκάρεται. Παρόλα αυτά συνιστάται να χρησιμοποιείται μία μόνο καρτέλα ανά browser profile.

Τα backups περιέχουν προσωπικά και οικονομικά δεδομένα. Αποθηκεύστε τα σε ασφαλή τοποθεσία και αποφύγετε την κοινή χρήση τους χωρίς υπηρεσιακή ανάγκη.

Η διαγραφή browser data, αλλαγή browser profile ή καθαρισμός `localStorage` μπορεί να προκαλέσει απώλεια τοπικών δεδομένων.

## Εκτέλεση

Η εφαρμογή δεν χρειάζεται build ή εξωτερικές βιβλιοθήκες. Λόγω των JavaScript modules πρέπει να ανοίξει μέσω τοπικού web server.

```bash
python -m http.server 8000
```

Έπειτα άνοιξε το `http://localhost:8000`.
