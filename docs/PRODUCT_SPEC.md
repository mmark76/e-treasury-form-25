# e-Treasury Form 25 — Product Specification

## 1. Σκοπός

Η εφαρμογή χρησιμοποιείται για την ηλεκτρονική συμπλήρωση, αποθήκευση, προβολή, εκτύπωση και αναζήτηση τιμολογίων Γ.Λ.25, αντί της χειρόγραφης διαδικασίας.

Οι βασικές εργασίες είναι:

1. Έκδοση τιμολογίου.
2. Διαχείριση στοιχείων Τμήματος / Υπηρεσίας.
3. Διαχείριση αρχείου οφειλετών / πελατών.
4. Αναζήτηση και διαχείριση αρχείου τιμολογίων.
5. Προβολή του συμπληρωμένου επίσημου τιμολογίου.

## 2. Τίτλος εφαρμογής

Ο τίτλος της εφαρμογής είναι:

`e-Treasury Form 25`

Να χρησιμοποιείται με ακριβώς αυτή τη γραφή και κεφαλαιοποίηση.

## 3. Αρχική σελίδα

Η αρχική σελίδα πρέπει να είναι απλή και υπηρεσιακή, χωρίς γενικό μήνυμα τύπου «Καλωσήρθατε».

Να εμφανίζει πέντε βασικές κάρτες:

1. `Έκδοση Τιμολογίου`
2. `Ρυθμίσεις Τμήματος / Υπηρεσίας`
3. `Αρχείο Οφειλετών / Πελατών`
4. `Αρχείο Τιμολογίων`
5. `Προβολή Τιμολογίου`

Επάνω δεξιά να υπάρχει ξεχωριστό κουμπί:

`Ρυθμίσεις Σελίδας`

### Οπτική κατεύθυνση

- Σκούρα μπλε κεφαλίδα.
- Ζεστό υπόλευκο φόντο.
- Ήπιες ζεστές αποχρώσεις στις κάρτες και στα εικονίδια:
  - κεχριμπαρί,
  - τερακότα,
  - φασκομηλί,
  - πετρόλ.
- Καθαρή, υπηρεσιακή και προσβάσιμη εμφάνιση.
- Responsive διάταξη.
- Πλήρης λειτουργία με πληκτρολόγιο.
- Όχι περιττά στατιστικά ή charts στην αρχική σελίδα.

## 4. Κύρια πλοήγηση

Η σειρά των βασικών επιλογών είναι:

1. Έκδοση Τιμολογίου
2. Ρυθμίσεις Τμήματος / Υπηρεσίας
3. Αρχείο Οφειλετών / Πελατών
4. Αρχείο Τιμολογίων
5. Προβολή Τιμολογίου

Οι κάρτες είναι επιλογές πλοήγησης και όχι κουμπιά ενεργειών.

## 5. Έκδοση Τιμολογίου

Η καθημερινή φόρμα πρέπει να περιλαμβάνει μόνο τα αναγκαία πεδία.

### 5.1 Βασικά στοιχεία

- Αριθμός τιμολογίου
- Ημερομηνία έκδοσης
- Επιλογή από Αρχείο Οφειλετών / Πελατών
- Όνομα Οφειλέτη / Πελάτη
- Χώρος / Υποστατικό που παραχωρείται

Η «Επιλογή από Αρχείο Οφειλετών / Πελατών» είναι εργαλείο αναζήτησης και επιλογής αποθηκευμένου πελάτη.

Το «Όνομα Οφειλέτη / Πελάτη» είναι το πραγματικό πεδίο που εμφανίζεται στο τιμολόγιο.

Με την επιλογή πελάτη να συμπληρώνονται αυτόματα:

- το όνομα,
- ο χώρος / υποστατικό.

Τα δύο πεδία να παραμένουν επεξεργάσιμα για το συγκεκριμένο τιμολόγιο.

### 5.2 Παροχή υπηρεσίας

Υποστηριζόμενα είδη:

- Παραχώρηση χώρου
- Κατανάλωση ηλεκτρικού ρεύματος
- Συνδυασμός των δύο
- Άλλη υπηρεσία, όπου απαιτείται

Πεδία:

- Είδος υπηρεσίας
- Μήνας / περίοδος
- Περιγραφή παροχής

Η περιγραφή να μπορεί να δημιουργείται αυτόματα από το είδος υπηρεσίας και την περίοδο, αλλά να παραμένει επεξεργάσιμη.

### 5.3 Ποσά

Ο χρήστης καταχωρίζει:

- Καθαρό ποσό, πριν από Φ.Π.Α.
- Συντελεστή Φ.Π.Α.

Η εφαρμογή υπολογίζει αυτόματα:

- ποσό Φ.Π.Α.,
- συνολικό ποσό,
- ποσό ολογράφως.

### 5.4 Έκδοση

- Ονοματεπώνυμο εκδότη
- Ημερομηνία υπογραφής

Η ημερομηνία έκδοσης στο επάνω και κάτω μέρος του επίσημου εντύπου πρέπει να ενημερώνεται σωστά από τα συμφωνημένα πεδία.

### 5.5 Ενέργειες

Τα κουμπιά ενεργειών είναι:

- `Καθαρισμός`
- `Αποθήκευση Πρόχειρου`
- `Αποθήκευση Τελικού Τιμολογίου`
- `Εκτύπωση`
- `Λήψη PDF`

#### Αποθήκευση Πρόχειρου

- Αποθηκεύει την τρέχουσα εργασία.
- Επιτρέπει ελεύθερη μεταγενέστερη επεξεργασία.
- Δεν θεωρείται εκδοθέν τιμολόγιο.

#### Αποθήκευση Τελικού Τιμολογίου

- Ελέγχει τα υποχρεωτικά πεδία.
- Αποθηκεύει το τιμολόγιο στο αρχείο τελικών τιμολογίων.
- Ζητά επιβεβαίωση πριν από την τελική αποθήκευση.
- Δεν επιτρέπει ελεύθερη αντικατάσταση ή διαγραφή μετά την έκδοση.
- Μελλοντικά πρέπει να υποστηρίζει ακύρωση ή διορθωτική εγγραφή αντί για διαγραφή.

## 6. Ρυθμίσεις Τμήματος / Υπηρεσίας

Εδώ αποθηκεύονται τα στοιχεία που συνήθως παραμένουν σταθερά:

- Τμήμα / Υπηρεσία
- Κωδικός Κεφαλαίου
- Αριθμός Μητρώου Φ.Π.Α.
- Διεύθυνση Υπηρεσίας
- Ταχυδρομικός κώδικας
- Λογαριασμός Εσόδων Προϋπολογισμού
- Προεπιλεγμένος συντελεστής Φ.Π.Α.
- Προεπιλεγμένο ονοματεπώνυμο εκδότη

Τα στοιχεία αυτά να συμπληρώνουν αυτόματα τη φόρμα έκδοσης και να μην καταλαμβάνουν μεγάλο καθημερινό panel.

## 7. Αρχείο Οφειλετών / Πελατών

Κύριες λειτουργίες:

- Προσθήκη νέου οφειλέτη / πελάτη
- Αναζήτηση
- Επεξεργασία
- Επιλογή για νέο τιμολόγιο

Βασικά αποθηκευμένα στοιχεία:

- Όνομα Οφειλέτη / Πελάτη
- Χώρος / Υποστατικό που παραχωρείται

Πρόσθετα στοιχεία να εμφανίζονται μόνο αν είναι πραγματικά αναγκαία.

## 8. Αρχείο Τιμολογίων

Το αρχείο περιλαμβάνει δύο εσωτερικές κατηγορίες:

- Πρόχειρα
- Τελικά

### Φίλτρα αναζήτησης

- Αριθμός τιμολογίου
- Όνομα οφειλέτη / πελάτη
- Μήνας / περίοδος
- Από ημερομηνία
- Έως ημερομηνία
- Είδος υπηρεσίας

Ο μήνας / περίοδος και το είδος υπηρεσίας πρέπει να αποθηκεύονται ως ξεχωριστά δεδομένα και όχι να εξάγονται μόνο από την περιγραφή.

### Ενέργειες αποτελέσματος

- Προβολή
- Εκτύπωση
- Λήψη PDF

Τα τελικά τιμολόγια δεν πρέπει να διαγράφονται ή να αντικαθίστανται ελεύθερα.

## 9. Προβολή Τιμολογίου

Η οθόνη εμφανίζει το συμπληρωμένο επίσημο έντυπο Γ.Λ.25 όπως θα εκτυπωθεί ή θα αποθηκευτεί σε PDF.

Χρησιμοποιείται για έλεγχο:

- ορθότητας στοιχείων,
- ποσών και Φ.Π.Α.,
- ημερομηνιών,
- περιγραφής,
- διάταξης,
- περικοπών κειμένου.

Να αντιμετωπιστεί το σημερινό κόψιμο σε:

- μεγάλα ονόματα οφειλετών,
- μεγάλες ονομασίες υπηρεσιών,
- διευθύνσεις,
- περιγραφές παροχής.

Η προβολή να ενημερώνεται από τα τρέχοντα δεδομένα της φόρμας.

## 10. Ρυθμίσεις Σελίδας

Το κουμπί εμφανίζεται επάνω δεξιά και αφορά γενικές ρυθμίσεις εφαρμογής, όχι στοιχεία τιμολογίου.

Πιθανές επιλογές:

- Μέγεθος προβολής
- Φωτεινή / σκοτεινή εμφάνιση
- Αυτόματη αποθήκευση πρόχειρου
- Επιβεβαίωση πριν από καθαρισμό
- Εξαγωγή αντιγράφου ασφαλείας
- Εισαγωγή αντιγράφου ασφαλείας
- Επαναφορά προεπιλεγμένων ρυθμίσεων

## 11. Απλοποίηση σημερινής φόρμας

Να αφαιρεθούν από την καθημερινή ροή ή να μετακινηθούν:

- τα per-field κουμπιά προτύπων,
- η διπλή καταχώριση στοιχείων οφειλέτη,
- τα μεγάλα panels σταθερών στοιχείων υπηρεσίας,
- τα panels αρχείου τιμολογίων και πελατών από την κύρια φόρμα.

Η υπάρχουσα λειτουργικότητα να μην διαγραφεί πριν μεταφερθεί και δοκιμαστεί στη νέα θέση.

## 12. Συμβατότητα και διατήρηση δεδομένων

Κατά τη σταδιακή μετάβαση:

- να διατηρηθούν τα υπάρχοντα `localStorage` keys,
- να διατηρηθούν τα αποθηκευμένα δεδομένα,
- να διατηρηθούν τα ενεργά field IDs όπου είναι δυνατό,
- να μην αλλάξει η αριθμητική λογική,
- να μην αλλάξει η παραγωγή PDF χωρίς ξεχωριστό εγκεκριμένο PR,
- να μην αλλάξει το επίσημο template χωρίς ξεχωριστό έλεγχο.
