# E-Treasury Form 25 — Audit Backlog

> Σκοπός: καταγραφή των ευρημάτων του ελέγχου του repository.  
> Τα ευρήματα πρέπει να επαληθεύονται στον τρέχοντα κώδικα πριν από κάθε αλλαγή.  
> Να υλοποιούνται σε μικρά, ανεξάρτητα PRs και όχι όλα μαζί.

## P0 — Κρίσιμα

1. Το «αρχείο εκδοθέντων» επιτρέπει αντικατάσταση και διαγραφή τιμολογίων χωρίς ιστορικό.
2. Δεν υπάρχει audit trail: χρήστης, λόγος αλλαγής, παλιά έκδοση και ημερομηνία μεταβολής.
3. Όλα τα δεδομένα αποθηκεύονται μόνο στο `localStorage` του browser.
4. Δεν υπάρχουν backup, συγχρονισμός, ανάκτηση ή κεντρική βάση δεδομένων.
5. Δεν υπάρχουν χρήστες, ρόλοι, δικαιώματα ή έλεγχος πρόσβασης.
6. Προσωπικά και φορολογικά στοιχεία αποθηκεύονται τοπικά χωρίς κρυπτογράφηση.
7. Αποτυχία αποθήκευσης μπορεί να αγνοηθεί και το UI να εμφανίσει επιτυχία.
8. Η φόρτωση ή εκτύπωση παλιού τιμολογίου μπορεί να αντικαταστήσει το τρέχον draft.
9. Η μετατροπή ποσού ολογράφως περιορίζεται σιωπηρά στα €999.999.
10. Υπάρχουν λανθασμένες γραμματικές αποδόσεις σε ορισμένες εκατοντάδες χιλιάδες.
11. Δεν έχει πιστοποιηθεί επίσημα η ακριβής ταύτιση του template με το εγκεκριμένο Γ.Λ.25.

## P1 — Υψηλή προτεραιότητα

12. Ελάχιστα πεδία είναι πραγματικά υποχρεωτικά.
13. Μπορεί να εκδοθεί έντυπο χωρίς πελάτη, περιγραφή, λογαριασμό ή με μηδενικό ποσό.
14. Δεν υπάρχει ουσιαστική επικύρωση φορολογικής ταυτότητας, τηλεφώνου και ταχυδρομικού κώδικα.
15. Ο αριθμός τιμολογίου κρατά μόνο τα τελευταία πέντε ψηφία, χωρίς προειδοποίηση.
16. Η μοναδικότητα τιμολογίου ελέγχεται μόνο από τον αριθμό, χωρίς έτος ή σειρά.
17. Δεν υπάρχει κεντρική ή ασφαλής αυτόματη αρίθμηση.
18. Δεν υπάρχουν καταστάσεις όπως draft, issued, cancelled ή corrected.
19. Δεν υπάρχει διαδικασία πιστωτικού ή διορθωτικού παραστατικού.
20. Η αντιστοίχιση πελάτη γίνεται ακόμη και μόνο από το όνομα και μπορεί να συνδέσει λάθος πρόσωπο.
21. Η διαγραφή πελάτη μπορεί να αφήσει τιμολόγια χωρίς προσβάσιμη καρτέλα.
22. Δεν υπάρχει referential-integrity έλεγχος μεταξύ πελατών και τιμολογίων.
23. Δεν υπάρχει export/import ολόκληρου του αρχείου.
24. Δεν υπάρχει μηχανισμός migration για αλλαγές στη δομή αποθηκευμένων δεδομένων.
25. Κατεστραμμένα δεδομένα `localStorage` αντικαθίστανται από fallback χωρίς ουσιαστική ανάκτηση.
26. Τα πλήρη snapshots επαναλαμβάνονται και μπορούν να εξαντλήσουν το όριο του browser.
27. Δεν υπάρχει pagination ή διαχείριση μεγάλου αριθμού εγγραφών.
28. Η ταξινόμηση του αρχείου δεν είναι καθορισμένη.
29. Μη έγκυρες ημερομηνίες φίλτρων αγνοούνται χωρίς εμφανές μήνυμα.
30. Δεν υπάρχουν όρια για υπερβολικά ποσά ή συντελεστές Φ.Π.Α.
31. Η αριθμητική ανάλυση μπορεί να αποδεχθεί μερικώς κακοσχηματισμένες τιμές εκτός του HTML input.

## P1 — PDF και εκτύπωση

32. Το PDF είναι ουσιαστικά μία JPEG εικόνα μέσα σε PDF.
33. Το κείμενο δεν είναι searchable, selectable ή προσβάσιμο.
34. Δεν υπάρχει text wrapping στον canvas renderer.
35. Μεγάλα ονόματα, διευθύνσεις και περιγραφές μπορεί να κοπούν.
36. Η υψηλή ανάλυση και ποιότητα JPEG καταναλώνουν σημαντική μνήμη.
37. Η εικόνα μπορεί να υποβαθμίσει τη σαφήνεια γραμμών και μικρών γραμμάτων.
38. Δεν υπάρχουν PDF metadata, PDF/A ή άλλες αρχειακές προδιαγραφές.
39. Δεν υπάρχει ψηφιακή υπογραφή ή έλεγχος αυθεντικότητας.
40. Εκτύπωση browser και ληφθέν PDF μπορεί να έχουν διαφορετική εμφάνιση.
41. Τα κουμπιά PDF μέσα στους πίνακες δεν εμφανίζουν κατάλληλο μήνυμα όταν αποτύχει η παραγωγή.
42. Δεν υπάρχει αυτοματοποιημένος έλεγχος ότι το PDF ανοίγει και έχει σωστή διάσταση Α4.

## P2 — Αρχιτεκτονική και κώδικας

43. Υπάρχουν δύο παράλληλες υλοποιήσεις προεπισκόπησης.
44. Το `index.html` περιέχει παλιό SVG overlay που αντικαθίσταται από JavaScript.
45. Το `preview.js` φαίνεται πλέον ανενεργό.
46. Παραμένουν παλιές CSS κλάσεις και ανενεργό SVG asset.
47. Φορτώνεται περιττά το SVG πριν αντικατασταθεί από το PNG.
48. Υπάρχει διπλή λογική για ημερομηνίες, filenames, escaping και δημιουργία κουμπιών.
49. Το CSS εκτύπωσης επαναλαμβάνεται σε περισσότερα αρχεία.
50. Υπάρχει unused function `formatDate` στο ενεργό template module.
51. Το `servicePostalCode` δημιουργείται δυναμικά αντί να υπάρχει στο HTML.
52. Τα features εξαρτώνται έντονα από global DOM selectors.
53. Υπάρχουν άμεσες εξαρτήσεις μεταξύ customers, invoice archive και PDF.
54. Δεν υπάρχουν καθαρά repository ή service interfaces για τα δεδομένα.
55. Τα `customers/ui.js` και `invoice-archive/ui.js` είναι υπερβολικά μεγάλα.
56. Το `app.css` έχει γίνει μεγάλο και περιέχει πολλές διαφορετικές ευθύνες.
57. Η πραγματική δομή αποκλίνει από τον ίδιο τον αρχιτεκτονικό οδηγό του project.
58. Η φόρτωση snapshot προκαλεί πολλά διαδοχικά input/change events και επαναλαμβανόμενα renders.
59. Η προεπισκόπηση επανυπολογίζεται τόσο στο `input` όσο και στο `change`.
60. Οι απόλυτες CSS συντεταγμένες είναι εύθραυστες σε αλλαγή template.

67. Δεν υπάρχουν integration tests.

69. Δεν υπάρχουν PDF tests.

## P2 — Προσβασιμότητα και UX

78. Το ενεργό overlay είναι `aria-hidden`, άρα screen reader δεν διαβάζει τα συμπληρωμένα στοιχεία.
79. Η προεπισκόπηση εμφανίζεται ουσιαστικά ως εικόνα.
80. Οι πίνακες δεν έχουν captions και πληρέστερη σημασιολογική σήμανση.
81. Η εφαρμογή βασίζεται σε browser `alert` και `confirm`.
82. Δεν υπάρχουν μη παρεμβατικά status, success και error messages.
83. Δεν υπάρχει undo μετά από διαγραφή.
84. Δεν υπάρχει προειδοποίηση για μη αποθηκευμένες αλλαγές.
85. Δεν εμφανίζεται διαθέσιμος χώρος αποθήκευσης ή κίνδυνος quota.
86. Η οριζόντια κύλιση των πινάκων δυσκολεύει τη χρήση σε κινητό.
87. Δεν υπάρχει πλήρης keyboard ή accessibility πιστοποίηση.

88. Το README περιγράφει παλιότερη και ελλιπή δομή.

## Προτεινόμενη σειρά εργασίας

1. Αριθμητικά σφάλματα, ποσά ολογράφως και αρίθμηση.
2. Επικύρωση πριν από έκδοση, αποθήκευση ή PDF.
3. Αμεταβλητότητα αρχείου και προστασία του τρέχοντος draft.
4. Αξιόπιστη αποθήκευση, migrations και χειρισμός σφαλμάτων.
5. PDF, clipping και καθαρισμός της διπλής προεπισκόπησης.
6. CI, tests, accessibility και τεκμηρίωση.

## Κανόνας υλοποίησης

Κάθε εργασία πρέπει να:
- επαληθεύει πρώτα το εύρημα στον τρέχοντα κώδικα,
- περιορίζεται σε ένα μικρό και σαφές scope,
- περιλαμβάνει tests και acceptance criteria,
- αποφεύγει άσχετα refactors,
- μην κάνει commit, push ή merge χωρίς ρητή έγκριση.
